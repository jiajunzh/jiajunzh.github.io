CS61B-2: Bits, Type, Lists, Sets

Posted on 2019-07-06 | Post modified: 2019-07-06 | In CS61B , Lecture Notes

Arrays have a fixed size ?
Then come Lists !

Bits

Memory is stored as a sequence of 0 and 1 and bits Interpreted using types

  • Variable Types
    • 8 primitive types: byte, short, int, long, float, double, char, boolean
    • Reference Types
  • Golden Rule of Equals
    x = y => copy the bits from y to x
    Parameter Passing => just copy the bits (pass by value always)

Object Instantiation

  • Declaring a Variable (primitive type)
    1. Set aside memory (Java will set aside exact bits to hold a thing of type)
    2. Create an entry in an internal table that maps variable name to the location of the first bit in the box by Java Interpreter
    ☑️ Java does not write anything to reserved boxes when declaring variables (no default values)
    ☑️ Java does not allow accessing a variable uninitialized

  • Reference Variables declaration
    1. Allocate a box of 64 bits
    2. Address of instance returned by new operator and bits copied into box
    Null == all zeros

  • Using new keyword
    1. Allocates a box for each instance variable of the class and fills them with a default value.
    2. Constructor then usually fills every box with some other value
    (In real Java, there is additional overhead for any object)

  • Arrays Instantiation

    • Arrays Structure
      • A fixed integer length N
      • A sequence of N memory boxes where all boxes are of the same type (0 thru N - 1)
    • Array Creation
    • Copy Array Java Arrays only perform bounds checking at runtime
    • 2D Arrays – Array of Array References

Basic Int Linked List

 Naked recursive data structure

Singly Linked List (SLList)

  • Improvement #1: Rebranding
     Rename IntList class to IntNode class
  • Improvement #2: Bureaucracy
     IntNode is hard to work with => create a separate class called SLList
     SLList as a middle man between user & naked recursive data structure

  • Improvement #3: Public vs. Private
     Declare first variable of SLList class as private
      => inaccessible naked data structure
  • Improvement #4: Nested Classes
     Embed a class declaration inside of another
    • Static Nested classes
      When nested class never use any instance members of the outer class => can not access any of the members of the enclosing class
      Save small amount of memory
  • Improvement #5: addLast(), size() and Caching
    • Recursive Approach of size()
      Slow method for long list (scan thru the whole list)
    • OverLoaded Method
      Two methods with the same name but different signatures
    • Caching
      Add a size variable to track the current size
      Increase memory usage and slow down add method
  • Improvement #6: The Empty List
     Constructor to create an empty list (first = null; size = 0)
      => addLast Null Pointer Excep
     SLList with Sentinel Node

Doubly Linked List (DLList)

  • Limitations of SLList and Fix
    Limitation and Reason => Slow method for long list (scan thru the whole list)
    Approach #1: add a last variable
     Limitations: addLast () and getLast () will be fast but removeLast () will be
     slow (to get second-to-last node)

   Approach #2: Looking Back – Doubly Linked List
   Add a previous pointer to each node, 2 links for each node

   Limitation: last pointer sometimes points at the sentinel node and sometimes a
   real node => getLast() method wrong

  • Improvements: Sentinel Upgrade
    Approach #1: add a second sentinel node

   Approach #2: Circular List
   front and back pointer sharing the same sentinel node

Generic Lists (e.g. DLLists)

  • Limitations of the Above Lists
    They can only hold integer values
  • Generics
    • Declaration Syntax
    • Instantiation Syntax
    • Syntax Details
      ☑️Generics only work with reference type;
      ☑️Empty diamond operator can be used during instantiation if declaration happens at the same line (but can also declare the type inside of the angle brackets during instantiating)

ArrayList

  • Linked List Performance of get(i)
    Linear in the size of the entire list
    Slow for random access => use array instead
  • Array Resizing
    Approach #1: add more size at once => slow
    Approach #2: multiply size by 2 (scaling factor) – Geometric Resizing
  • Improvement: Spike of events
    Usage ratio = size of the list / length of the array
    Half the size when ratio < 0.25

  • Generic AList Nulling out deleted items for generics objects to avoid loitering (Java only destroys (garbage collects) object when the last reference has been lost)

  • Set
